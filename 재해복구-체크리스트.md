# SG-LOK 재해복구 체크리스트

노트북 고장/분실/포맷 상황에서도 개발을 즉시 재개하기 위한 체크리스트입니다.

## 1) 평소 운영 체크리스트 (사전 예방)

- [ ] 모든 코드 변경사항을 GitHub 원격 저장소에 자주 `push`한다.
- [ ] `main` 브랜치는 항상 배포 가능한 상태로 유지한다.
- [ ] 기능 개발은 작업 브랜치에서 진행 후 병합한다.
- [ ] `.env.local` 파일은 Git에 커밋하지 않고, 비밀관리 도구(비공개)에 별도 보관한다.
- [ ] Firebase 콘솔 접근 계정(2FA 포함) 및 권한을 최신 상태로 유지한다.
- [ ] Vercel 프로젝트 연결 상태 및 환경변수를 정기 점검한다.

## 3) 장애 발생 시 즉시 복구 절차 (새 노트북 기준)

### A. 개발 환경 준비
- [ ] Git 설치
- [ ] Node.js 설치 (프로젝트 권장 버전)
- [ ] 패키지 매니저 준비 (`npm`)
- [ ] Cursor/에디터 설치

### B. 프로젝트 복원
- [ ] 저장소 클론  
  - `git clone <저장소 URL>`
- [ ] 프로젝트 폴더 진입
- [ ] 의존성 설치  
  - `npm install`
- [ ] `.env.local` 복원 (비밀관리 도구에서 값 복사)
- [ ] 개발 서버 실행  
  - `npm run dev`

### C. 외부 서비스 연결 확인
- [ ] Firebase 인증/Firestore 연결 정상 여부 확인
- [ ] 로그인/회원/관리자 페이지 동작 확인
- [ ] Vercel 배포 연결 및 환경변수 동기화 확인
